## 正文

最近重度用 Claude Code,有天好奇:**我这一个月到底花了多少?**

翻设置没有,翻官网也只有个模糊的用量。于是我顺手写了个小工具,直接读本地 `~/.claude/projects` 里的 session 记录(只读 token 计数,不碰对话内容、不联网),把账单算了出来。结果有三件事挺反直觉:

**1. "用量价值" ≠ "你花的钱"。**

工具按 API 牌价一算:这个月我消耗了约 ¥1,800 的 token。我吓一跳——我明明只付了个 Pro 订阅啊。

后来想明白了:订阅是**固定月费**,这 ¥1,800 是"如果按 API 付费要花这么多"的**用量价值**,不是我真掏的钱。换句话说,我的订阅这个月**回本了 11 倍**。

> 所以工具现在会问你用的是订阅还是 API Key:订阅就给你算"回本几倍",API 才算"真金白银花了多少"。别被一个吓人的数字误导。

[配图:账单卡片 —— 真实支出 ¥184 / 订阅薅到价值 ¥1,851 · 回本 11.3x]

**2. 钱其实花在你看不见的地方:缓存重放。**

我以为花费 ≈ 我打的字。结果最大的一块是**缓存读取**——每轮对话都把整个上下文重放一遍,一个月下来是 **7,000 万 token**。虽然单价只有新鲜输入的 1/10,但量大得惊人。

好消息是我的缓存命中率有 93%(命中越高越省)。工具会告诉你你的命中率,以及如果偏低能怎么救。

**3. 模型分层是真省钱。**

我把一部分批量、不需要顶配的任务分流给了 DeepSeek,342 次调用只花了约 ¥21。同样的活儿全用 Opus 的话,得贵几十倍。这部分是按量付费,省的是真钱。
